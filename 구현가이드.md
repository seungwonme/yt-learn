# YouTube 학습 플랫폼 만들기

> AI와 함께 코딩 없이 만드는 학습 플랫폼 (2025년 최신 버전)

## 시작하기 전에

이 가이드는 **AI 어시스턴트(Claude, ChatGPT 등)** 와 함께 YouTube 학습 플랫폼을 만드는 방법입니다.
프로그래밍 지식이 없어도 AI에게 요청하는 방식으로 따라할 수 있습니다.

### 필요한 것

1. **터미널 (명령창)** - 맥의 터미널 또는 윈도우의 PowerShell
2. **AI 어시스턴트** - Claude Code, ChatGPT, Cursor 등
3. **API 키** - 나중에 발급받을 예정

## 📦 1단계: 프로젝트 초기 설정

### 1-1. UI 라이브러리 설치

터미널에서 실행:
```bash
pnpm dlx shadcn@latest init -d
```

**결과**: 예쁜 UI 컴포넌트들이 자동으로 설치됩니다.

### 1-2. 필요한 도구들 설치

터미널에서 실행:
```bash

```

**결과**: YouTube와 AI 기능을 사용하기 위한 도구들이 설치됩니다.

### 2-2. 자막 추출 기능 ⚠️ 중요 업데이트

**AI에게 이렇게 말하세요:**

```
YouTube 자막 추출 기능을 만들어줘.

기능:
- YouTube 영상에서 자막 추출
- youtubei.js 라이브러리의 Innertube API 사용
- 배포 환경(Vercel)에서도 안정적으로 작동

구현 방법:
1. Innertube.create()로 클라이언트 생성
2. youtube.getInfo(videoId)로 영상 정보 가져오기
3. info.getTranscript()로 자막 추출
4. transcript.content.body.initial_segments에서 자막 데이터 접근

중요 - 데이터 변환:
- start_ms와 end_ms는 밀리초(ms) 단위로 반환됨
- 초(seconds)로 변환 필요:
  - start: segment.start_ms / 1000
  - duration: (segment.end_ms - segment.start_ms) / 1000
  - text: segment.snippet.text

참고: npm에서 youtubei.js 문서 확인
```

**원하는 결과**:
- 영상 자막 추출 기능 완성
- 배포 환경에서도 정상 작동
- 타임스탬프가 올바르게 표시됨

## 🔑 API 키 발급하기

### YouTube Data API

1. @https://console.cloud.google.com/apis/credentials 접속
2. "프로젝트 만들기" 클릭
3. "사용자 인증 정보" → "API 키" 생성
4. YouTube Data API v3 활성화
5. API 키 복사

### Google Gemini API

1. @https://makersuite.google.com/app/apikey 접속
2. "API 키 만들기" 클릭
3. API 키 복사

### API 키 설정

1. 프로젝트 폴더에 `.env.local` 파일 생성
2. 내용 입력:
```
YOUTUBE_API_KEY=여기에_유튜브_API_키_붙여넣기
GOOGLE_GEMINI_API_KEY=여기에_제미나이_API_키_붙여넣기
YOUTUBE_MAX_RESULTS=10
```

7. ✅ 자막 클릭 → 해당 시간으로 이동
8. ✅ 타임스탬프 클릭 → 해당 위치로 이동
9. ✅ "AI 요약 재생성" 버튼으로 재생성 가능

## 🛠️ 문제 해결 가이드

### 자막 타임스탬프가 0:00으로 표시될 때

**원인**: `youtubei.js`는 start_ms와 end_ms를 밀리초로 반환

**해결**: AI에게 이렇게 말하세요

```
자막 시간이 밀리초로 표시되고 있어.
segment.start_ms와 segment.end_ms를 1000으로 나눠서 초 단위로 변환해줘.

변환 방법:
- start: segment.start_ms / 1000
- duration: (segment.end_ms - segment.start_ms) / 1000
```

참고: npm의 youtubei.js 문서를 확인하세요.

### Zod 에러: "error.errors does not exist"

**원인**: Zod 4.x에서 `errors` 프로퍼티가 `issues`로 변경됨

**해결**: AI에게 이렇게 말하세요

```
Zod 에러 처리에서 error.errors를 error.issues로 바꿔줘.
Zod 4.x 버전을 사용하고 있어.
```

참고: Zod 공식 문서의 Error Handling 섹션을 확인하세요.

### Next.js: "useSearchParams should be wrapped in suspense"

**원인**: Next.js 15에서 useSearchParams를 Suspense로 감싸야 함

**해결**: AI에게 이렇게 말하세요

```
useSearchParams 대신 sessionStorage를 사용해서
검색 상태를 저장하고 복원하도록 바꿔줘.
```

참고: Next.js 15 공식 문서를 확인하세요.
